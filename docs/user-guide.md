# FFCodec Lab 用户指南

> v0.5.1 — FFmpeg 压制命令生成器

---

## 快速开始

1. 安装 Node.js 20+，执行 `npm install` 与 `npm run dev`
2. 左侧面板配置编码参数
3. 右侧实时预览生成的 FFmpeg 命令
4. 点击「复制」按钮将命令复制到剪贴板
5. 在终端中粘贴运行

FFCodec 不调用 FFmpeg — 它只生成命令供你使用。

## 视频编码

### 编码器选择

FFCodec 支持 11 种视频编码器：

**软件编码器**（不需要特定硬件）：
- **libx264** — H.264/AVC，兼容性最好
- **libx265** — H.265/HEVC，压缩效率更高
- **libsvtav1** — AV1，最新一代编码

**硬件编码器**（需要特定 GPU）：
- **h264_nvenc** — NVIDIA GPU H.264 硬件编码
- **hevc_nvenc** — NVIDIA GPU H.265 硬件编码
- **h264_qsv** — Intel GPU H.264 硬件编码（Quick Sync Video）
- **hevc_qsv** — Intel GPU H.265 硬件编码
- **h264_amf** — AMD GPU H.264 硬件编码
- **hevc_amf** — AMD GPU H.265 硬件编码
- **h264_videotoolbox** — Apple 平台 H.264 硬件编码
- **hevc_videotoolbox** — Apple 平台 H.265 硬件编码

### 质量模式

不同编码器支持不同的质量控制模式：

| 模式 | 说明 | 适用编码器 |
|------|------|-----------|
| CRF | 恒定质量（推荐） | libx264, libx265, libsvtav1 |
| CQP | 恒定量化参数 | 所有编码器 |
| ICQ | 智能恒定质量（QSV 特有） | h264_qsv, hevc_qsv |
| LA_ICQ | 前瞻智能质量（QSV 特有） | h264_qsv, hevc_qsv |
| VBR | 可变码率 | 所有编码器 |
| CBR | 恒定码率 | 所有编码器 |
| CQ | 恒定质量（NVENC 特有） | h264_nvenc, hevc_nvenc |
| Two-Pass | 二次编码 | libx264, libx265 |

## 音频编码

- **AAC** — 最广泛兼容，适合 MP4
- **Opus** — 高效率音频编码，适合 MKV/WebM
- **FLAC** — 无损压缩，零质量损失

FLAC 无码率参数 — 压缩级别只影响文件大小和编码速度。

声道布局使用下拉选项，可选择跟随输入、mono、stereo、2.1、3.0、quad、5.1、5.1(side) 或 7.1。采样率可选择跟随输入，以及从 8000 Hz 到 192000 Hz 的常用档位。

## 输入流保留

在「输入与输出」区域，可以分别控制视频、音频和内置字幕：

- 开启「保留全部」时，该类型的全部输入流都会映射到输出。
- 关闭时，仅保留对应流索引选择的流。
- 将某类媒体的编码模式设为「禁用」仍会优先禁用该类输出。

---

## 画面处理

「画面参数」支持裁剪、90°/180° 旋转、水平/垂直镜像、亮度/对比度/饱和度/伽马调整、YADIF 去隔行、锐化、缩放和帧率调整。系统按固定顺序合成单一 `-vf` 滤镜链，避免参数互相覆盖。

---

## 字幕

### 字幕混流（不重编码视频）

在「字幕」区域添加字幕轨道：
1. 选择来源（输入文件中 / 外挂文件）
2. 选择编码方式（复制原始流 / 转码）
3. 设置语言、标题、默认轨道、强制字幕

### 字幕烧录（嵌入视频画面）

1. 开启「启用字幕烧录」
2. 选择字幕来源
3. 可选：字体、字号、粗斜体、颜色、描边、阴影、位置、边距和字距

字幕烧录要求视频重编码（video mode = encode）。

## 预设管理

点击右上角「💾 预设」按钮打开预设管理：

- **内置预设**：5 个常用配置（H.264 日常、H.265 高质量等）
- **用户预设**：你可以保存、编辑、导出自己的配置
- **导入/导出**：JSON 格式，方便分享和备份
- **恢复默认**：清除所有自定义，回到初始状态

预设只保存配置参数，不保存生成的命令文本。

## 命令预览

- **默认显示**：PowerShell 单行命令
- **三种 Shell**：Bash、PowerShell、CMD
- **多行模式**：每个参数独立一行，点击参数可定位到对应控件
- **单行模式**：紧凑格式，方便直接粘贴
- **错误时禁止复制**：配置有错误时复制按钮不可用

页面默认使用亮色模式；顶部「暗色模式」按钮可切换主题，选择会保存在当前浏览器中。

## 分享与高级参数

### 模块化参数工作台

参数不再全部堆在同一页。桌面端使用左侧导航切换输入与输出、视频编码、质量控制、色彩管理、画面与滤镜、音频、字幕、流与封装及自定义参数；右侧命令检查器始终显示当前命令或诊断。移动端可从顶部选择模块，并从底部打开命令检查器。

模块地址使用 `?panel=`，可以与分享配置的 `#...` 同时使用。点击命令片段或诊断定位时，页面会自动切换到对应模块。

### 高级质量、色彩与滤镜

- 质量控制页按当前编码器显示 GOP、B 帧、关键帧间隔和量化控制等可用项。
- 色彩管理页可选择“仅写入元数据”“写入元数据并转换”或“仅转换”。默认仍是只写标记，不改变旧配置行为。
- 实际转换可使用 zscale（CPU）或 libplacebo（GPU）；zscale 可组合 CPU tonemap 完成 HDR→SDR。libplacebo 是否可用取决于 FFmpeg 构建、Vulkan 与驱动。
- 色调映射必须明确目标传输特性；HDR 转 SDR 通常同时选择 bt709 矩阵、原色、传输特性和 tv 范围，并用短样片检查亮度与色偏。
- 画面与滤镜页提供 hqdn3d、nlmeans、atadenoise、bm3d、deband 和 gradfun，并与现有裁剪、缩放、调色、锐化等合并为一个有序 `-vf`。
- 高级参数留空时不会进入命令。可选布尔参数可选择“不设置”“开启”或“关闭”。
- 高级质量第二批包含适用编码器的 level、前瞻、AQ 强度、场景切换、参考帧、Intel extbrc 和 AMD QVBR 质量级别；切换编码器会清理旧编码器专属值。
- 参数来源主要参考 FFmpegFreeUI；实际可用性仍取决于本机 FFmpeg 构建、编码器、硬件和驱动。如遇错误，请记录 FFmpeg 版本、完整命令和错误输出以便修订。

- 「分享配置」把不含本地媒体路径的配置写入 URL Hash，可直接复制链接。
- 六个自定义参数区分别对应全局、输入前、视频、音频、输出前和命令末尾；每行填写一个 token。
- 自定义参数只做 Shell 转义，不验证 FFmpeg 语义，使用前请检查命令预览。

## 硬件编码器注意事项

FFCodec 不检测本机硬件。使用硬件编码器前请确认：

1. 运行 `ffmpeg -encoders`，确认存在所选的 nvenc、qsv、amf 或 videotoolbox 编码器
2. 安装了正确的 GPU 驱动
3. 硬件满足最低代际要求（参见编码器详情）

环境可用性提示只是提醒，不会阻止你复制命令。
