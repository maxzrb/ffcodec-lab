### 2026-07-10 14:30 — 首轮开发完成

- 本次目标：依据《Codex FFmpeg 命令生成器项目指令集》和《FFmpegFreeUI 参数树与网页首版裁剪方案》，建立项目骨架、数据目录、规则引擎、命令构建器和开发验证页面。
- 完成内容：
  - **项目配置**: Vite 5 + React 18 + TypeScript 严格模式 + Zustand + Zod + Vitest，共计 57 个源文件。
  - **核心类型**: `ProjectConfig`（10 个子配置块）、`EncoderDefinition`（含 preset/profile/tune/pix_fmt/qualityModes）、`ContainerDefinition`（兼容性矩阵）、`RuleExpression`（9 种声明式操作符）、`CommandPlan`/`CommandArg`（每个参数带 originId 和 phase）、`SubtitleMuxConfig` 与 `SubtitleBurnConfig` 完全分离。
  - **数据目录**: libx264（preset 10 档 / profile 7 档 / tune 9 档 / CRF 0~51 / CQP 0~69）、libx265（preset 10 档 / profile 5 档 / tune 8 档 / CRF 0~51）、libsvtav1（preset 8 档 [0~13] / profile 3 档 / CRF 0~63）、AAC（VBR/CBR / 码率 96k~320k）、libopus（VBR/CBR / 码率 64k~510k / application / frame_duration）。容器 MP4/MKV/WebM/MOV 含视频/音频/字幕三重兼容性矩阵。
  - **规则引擎**: 11 条规则覆盖 R01-R15（视频 copy 禁用质量控制/分辨率/烧录、视频 disabled 隐藏全部、音频 copy/disabled 对应行为、分辨率/帧率 vs copy 冲突、烧录 vs copy 冲突、MP4/MOV 字幕 auto→mov_text、WebM 字幕 auto→webvtt、编码器变化清除无效值、质量模式互斥、WebM + libx264/AAC 不兼容错误）。
  - **命令系统**: Command AST 含 15 阶段排序（GLOBAL→…→OUTPUT），从 Config 构建 CommandPlan 再经 Shell Renderer 输出。Bash/Zsh 用单引号转义，PowerShell 用双引号，CMD 用 ^ 转义。视频滤镜仅合并为一个 -vf 参数（scale→fps→subtitle burn）。
  - **验证系统**: 33 项单元测试（规则引擎 17、命令构建 11、规范化器 5），覆盖所有 R01-R15 不变量。目录审计脚本检查来源缺失、ID 重复、默认值范围、解释引用完整性。
  - **文档**: `docs/architecture.md`（分层架构图、数据流、15 个关键设计决策、完整文件清单）、`docs/source-map.md`（FFmpegFreeUI 源码→本项目模块逐一映射、5 项未核验参数清单）。
